    看法：

    因为之前我们消费者调用提供者有很高的耦合度，如果说中间出现异常或宕机就很难继续往下处理，但是出现了消息中间件以后就可以完美并轻易的解决这个问题。无论消费者还是提供者都只需要连接上消息队列后即可消费消息或者发送消息。无需知道对方的存在。这也是中间件的一大优点：解耦！

    二.Linux下RabbitMQ服务器搭建

    1.因为RabbitMQ使用Erlang语言开发的所以需要安装Erlang；

    安装erlang地址
otp_src_18.2.1.tar.gz 下载地址：
$ wget http://erlang.org/download/otp_src_18.2.1.tar.gz

    2.安装RabbitMQ地址

    rabbitmq-server-3.6.9.tar.xz 下载地址：
$ wget http://www.rabbitmq.com/releases/rabbitmq-server/v3.6.9/rabbitmq-server-generic-unix-3.6.9.tar.xz

    三.首先安装erlang
    1.安装GCC GCC-C++ Openssl等模块

    $ yum -y install make gcc gcc-c++ kernel-devel m4 ncurses-devel openssl-devel

    2.安装ncurses

    $  yum -y install ncurses-devel

    3.对Erlang源代码文件压缩

    $ tar -xvf otp_src_18.2.1.tar.gz

    4.然后，运行如下命令重命名文件夹为erlang。

    $ mv otp_src_18.2.1/  erlang

    5.检查编译环境并配置安装路径，运行如下命令：

    $ cd erlang/
$ ./configure --prefix=/usr/local/erlang --without-javac

    6.检查编译环境完之后进行编译安装

    $ make && make install

    7.设置环境变量添加Erlang环境

    $  vi  etc/profile
ERLANG_HOME=/usr/local/erlang
然后加入PATH中
PATH=$JAVA_HOME/bin:$ERLANG_HOME/bin:$PATH
这和windows上配置环境变量很相似，都是配完单个后，最后再加入PATH中进行生效。

    8.配置完后保存，并运行命令使环境变量立即生效

    $  source ~/.bash_profile     或者 #  source /etc/profile

    9.最后验证erlang是否安装成功。

    $ erl -version


    出现这个效果后说明Erlang安装成功，总结整体的步骤其实就四步：
    （1）.安装GCC GCC-C++ Openssl等模块
    （2）.检查编译环境并配置安装路径
    （3）.对源代码进行编译&&安装
    （4）.设置环境变量添加Erlang环境

    四.安装RabbitMQ
    1.因为RabbitMQ文件是xz文件所以第一步先下载xz压缩工具：

    $  yum install xz

    2.解压Rabbit压缩包：

    $ xz -d rabbitmq-server-generic-unix-3.6.9.tar.xz
$ tar -xvf rabbitmq-server-generic-unix-3.6.9.tar

    3.随后移到/usr/local/目录下改名为RabbitMq

    $ mv  rabbitmq_server-3.6.9 /usr/local/rabbitmq

    4.修改/etc/profile

    $ MQ_HOME=/usr/local/rabbitmq/sbin
执行source /etc/profile使得PATH路径更新，rabbitMQ安装成功。

    5.随后启动RabbitMQ管理方式

    $ cd /usr/local/rabbitmq/sbin
$ ./rabbitmq-plugins enable rabbitmq_management   #启动后台管理
$ ./rabbitmq-server -detached    #后台运行rabbitmq

    6.设置端口可供外部访问

    $  iptables -I INPUT -p tcp --dport 15672 -j ACCEPT

    7.因为rabbitMQ需要登录才能管理查看所以先创建用户

    $  ./rabbitmqctl add_user  admin 123456        ##  admin为账号， 123456为密码。

    8.授管理员权限

    $  ./rabbitmqctl set_user_tags admin administrator

    9.授权用户到虚拟机

    $  ./rabbitmqctl set_permissions -p "/" admin ".*" ".*" ".*"      也可以说是添加权限

    10.然后就可以远程访问了，然后可直接配置用户权限等信息。
    http://虚拟机ip:15672/

    到这里rabbitMQ搭建完成。 完结!告辞（靓仔挥手）！
