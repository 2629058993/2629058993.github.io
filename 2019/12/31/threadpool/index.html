    四大线程池

    java知识 thread pool 线程池 12月31日下午多云转晴
    创建时间:2019-12-31 16:08

    背景:

    1.线程每次new Thread()新建线程对象性能差。并且线程缺乏统一管理，可能无限制新建线程，相互之间
竞争,极可能占用过多系统资源导致系统死机。

2.缺乏很多功能，比如定时，周期性执行，或者线程中断。

3.所以再出现了种种问题之后就出现了线程池。

    线程池的好处:

    1.降低资源消耗，通过重复利用已创建的线程来降低重复创建线程、销毁线程造成的消耗

2.提高响应速度，当任务到达时，线程帮此任务省去了创建线程的时间。

3.提高线程的可管理性，线程是稀缺资源，如果无限制的创建，不仅会消耗系统资源，还会降低
系统的稳定性，使用线程池可以进行统一的分配、调优、分配。

    四大线程池：

    newFixedThreadPool:

    第一种比较常用newFixedThreadPool（固定大小的线程池,常用的哦）
    首先看一下这种线程池的创建方法（底层代码）：

    public static ExecutorService newFixedThreadPool(int nThreads) {
    return new ThreadPoolExecutor(nThreads, nThreads,
          0L, TimeUnit.MILLISECONDS,
          new LinkedBlockingQueue<Runnable>());

    }
    从这种构造方法可以看出，它创建了一个固定大小的线程池，每一次业务都会创建一个线程，直到到达线程池的最大值nThreads，线程池的大小一旦达到最大值之后，再有新任务提交时，就会将新提交的任务放入无界阻塞队列，等到有线程空闲时，再从队列中取出来继续执行任务。这就是newFixedThreadPool创建线程的大致流程。建议使用此线程池。
    例子如下：

    public class OneMoreStudy {
    public static void main(String[] args) {
    ExecutorService fixedThreadPool = Executors.newFixedThreadPool(3);

    for (int i = 0; i < 5; i++) {
        final int index = i;
        fixedThreadPool.execute(new Runnable() {
            @Override
            public void run() {
            try {
                SimpleDateFormat sdf = new SimpleDateFormat(
                    "HH:mm:ss");
                System.out.println("运行时间: " +
                sdf.format(new Date()) + " " + index);
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            }
        });
    }

    fixedThreadPool.shutdown(); 
    }
}

    newCachedThreadPool:

    第二种:newCachedThreadPool（缓冲线程池基于内存,线程个数不限制）
    首先看一下底层如何创建newCachedThreadPool这个对象：

    public static ExecutorService newCachedThreadPool() {
 return new ThreadPoolExecutor(0, Integer.MAX_VALUE,
                   60L, TimeUnit.SECONDS,
                   new SynchronousQueue<Runnable>());
 }

    从构造方法中可以看出，它创建了一个可缓存的线程池。当有新的任务提交时，有空闲线程就直接处理任务，没有空闲线程就创建线程，队列中于newFixedThreadPool这个线程池不同，它的队列不储存线程任务。并且此线程池没有线程限制，就是说只要有内存或者空闲线程都可以一直创建任务线程。线程的最大个数完全依赖于JVM能创建线程最大个数的大小。还有线程如果超过60秒空闲就会回收即删除。一般不建议使用此线程池，如果线程高发期，就会造成jvm内存溢出，线程池报错。
    例子如下：

    public class OneMoreStudy {
    public static void main(String[] args) {
    ExecutorService cachedThreadPool = Executors.newCachedThreadPool();

    for (int i = 0; i < 5; i++) {
        final int index = i;
        cachedThreadPool.execute(new Runnable() {
            @Override
            public void run() {
            try {
                SimpleDateFormat sdf = new SimpleDateFormat(
                    "HH:mm:ss");
                System.out.println("运行时间: " +
                sdf.format(new Date()) + " " + index);
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            }
        });
    }

    cachedThreadPool.shutdown();
    }
}

    newSingleThreadExecutor:

    第三种线程池newSingleThreadExecutor(线程池内部为单线程)
    首先，看一下此线程池的创建方法:

    public static ExecutorService newSingleThreadExecutor() {
    return new FinalizableDelegatedExecutorService
        (new ThreadPoolExecutor(1, 1,
                    0L, TimeUnit.MILLISECONDS,
                    new LinkedBlockingQueue<Runnable>()));
}

    从构造方法可以看出，他是一个单线程化的线程池，所谓单线程，就是每次执行的线程任务只有一个，只有这一个执行完之后才能执行下一个线程任务。保证了线程池内部所有线程任务井然有序。此线程池是单线程可能有人会疑惑，既然是单线程，那么这种线程池还有必要存在嘛？在此说明一下，这里的单线程指的是线程池内部，从线程池外部的角度看，主线程在提交任务到线程池时并没有堵塞。仍然是异步的。（我理解就是提交任务也就是发起线程这时候是很快的没有堵塞，但可能就是在执行线程任务时会按照顺序。效率较低）！

    例子如下：

    public class OneMoreStudy {
    public static void main(String[] args) {
    ExecutorService singleThreadExecutor = Executors.newSingleThreadExecutor();

    for (int i = 0; i < 5; i++) {
        final int index = i;
        singleThreadExecutor.execute(new Runnable() {
            @Override
            public void run() {
            try {
                SimpleDateFormat sdf = new SimpleDateFormat(
                    "HH:mm:ss");
                System.out.println("运行时间: " +
                sdf.format(new Date()) + " " + index);
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            }
        });
    }

    singleThreadExecutor.shutdown();
    }
}

    newScheduledThreadPool:

    第四种线程池newScheduledThreadPool（固定大小，支持定时，周期性任务执行）
    直接上例子：（首先定时例子：）

    public class OneMoreStudy {
    public static void main(String[] args) {
    final SimpleDateFormat sdf = new SimpleDateFormat("HH:mm:ss");
    ScheduledExecutorService scheduledThreadPool = Executors.newScheduledThreadPool(3);
    System.out.println("提交时间: " + sdf.format(new Date()));
    scheduledThreadPool.schedule(new Runnable() {
        @Override
        public void run() {
            System.out.println("运行时间: " + sdf.format(new Date()));
        }
        }, 3, TimeUnit.SECONDS);
    scheduledThreadPool.shutdown();
    }
}

    周期性任务执行例子：

    public class OneMoreStudy {
    public static void main(String[] args) {
    final SimpleDateFormat sdf = new SimpleDateFormat("HH:mm:ss");
    ScheduledExecutorService scheduledThreadPool = Executors.newScheduledThreadPool(3);
    System.out.println("提交时间: " + sdf.format(new Date()));
    scheduledThreadPool.scheduleAtFixedRate(new Runnable() {
        @Override
        public void run() {
            System.out.println("运行时间: " + sdf.format(new Date()));
        }
        }, 1, 3, TimeUnit.SECONDS);
    Thread.sleep(10000);
    scheduledThreadPool.shutdown();
    }
}

    使用该线程的scheduleAtFixedRate方法，含义就是延时一秒后每隔三秒执行任务，一直循环，我测试的时候不会停，就像循环无数次一样.运行结果如下:

    总结:本文共有四种常用线程池.
    (1).固定大小的线程池;超出最大值的线程任务放在线程队列,等待有线程空闲之后执行.(常用)
    (2).线程大小无限制的线程池.上限完全基于JVM内存的最大线程大小
    (3).单线程化的线程池,保证线程池内的任务执行顺序.线程池外部为异步,提交任务没有堵塞.单线程化值得是内部哦
    (4).固定大小,并且支持定时,以及周期性任务(和for循环差不多哈不会停的)
